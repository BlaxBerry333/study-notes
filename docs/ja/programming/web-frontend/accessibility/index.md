# Accessibility

Web アクセシビリティ -- 全てのユーザー（障害のあるユーザーを含む）が Web コンテンツを正常に利用できるようにする

::: warning 特徴

- **法的義務**: 日本では JIS X 8341-3 が基準、EU の EAA（2025 年施行）が義務化を推進、日本では 2024 年 4 月施行の改正障害者差別解消法により民間企業に合理的配慮の提供が義務化
- **WCAG ガイドライン**: W3C が策定した国際標準 WCAG 2.2 がデファクトスタンダード
- **4 原則（POUR）**: Perceivable（知覚可能）・Operable（操作可能）・Understandable（理解可能）・Robust（堅牢性）
- **開発での実践**: セマンティック HTML、WAI-ARIA、キーボード操作、色コントラストが主な実装ポイント

:::

```
WCAG 適合レベル

  Level A    ── 最低限（未達だと多くのユーザーが利用不可）
  Level AA   ── 標準的な目標（日本企業・公共サイトの実質的な基準）
  Level AAA  ── 最高（通常は全ページでの達成は求められない）
```

::: tip 日本の実務ポイント
**JIS X 8341-3:2016** は WCAG 2.0 Level AA に対応する。多くの日本企業は WCAG 2.1 AA を目標としている。2024 年 4 月施行の改正障害者差別解消法により、合理的配慮の提供が「努力義務」から「法的義務」に格上げされた
:::

## 基礎概念

| 概念                | 説明                                                                                                    |
| ------------------- | ------------------------------------------------------------------------------------------------------- |
| セマンティック HTML | 意味のある HTML 要素を正しく使い、支援技術がページ構造を理解できるようにする（[詳細](./semantic-html)） |
| WAI-ARIA            | HTML では伝えられない情報を補完する属性体系（[詳細](./aria)）                                           |
| キーボード操作      | マウスなしでも全機能にアクセスできる（[詳細](./keyboard)）                                              |
| フォーム            | ラベル、エラー、バリデーション情報を支援技術に正しく伝達する（[詳細](./form)）                          |
| テスト              | 自動テスト + 手動テスト + スクリーンリーダーの三層で品質を保証する（[詳細](./testing)）                 |
| 色とコントラスト    | テキストと背景のコントラスト比 4.5:1 以上（AA 基準）                                                    |
| レスポンシブ        | 200% ズーム時にコンテンツが欠損せず、reflow が正常に動作する                                            |
| 代替テキスト        | 画像に適切な `alt` 属性を設定し、装飾的な画像は `alt=""` でスキップする                                 |

## 基本的な使い方

アクセシビリティ対応の基本は**セマンティック HTML を正しく書くこと**である。多くの問題はセマンティック HTML（[詳細](./semantic-html)）だけで解決できる

不足する部分を WAI-ARIA（[詳細](./aria)）で補い、キーボード操作（[詳細](./keyboard)）とフォーム（[詳細](./form)）の対応を行い、最後にテスト（[詳細](./testing)）で品質を確認する

### チェックリスト

```
✅ 全ての画像に適切な alt がある
✅ 見出しレベルが正しい（h1 → h2 → h3、飛ばさない）
✅ フォーム入力に関連付けられた <label> がある
✅ Tab キーで全てのインタラクティブ要素にアクセスできる
✅ フォーカスインジケータが可視である
✅ 色だけで情報を伝えていない
✅ テキストのコントラスト比が 4.5:1 以上
✅ ページに lang 属性が設定されている
✅ エラーメッセージと入力欄がプログラム的に関連付けられている
```
