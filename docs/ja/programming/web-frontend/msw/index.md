# MSW

> Mock Service Worker

ネットワーク層でリクエストをインターセプトし、モックレスポンスを返す API モックツール

::: warning 特徴:

- ネットワーク層でインターセプト（Service Worker / Node.js インターセプター）するため、**ビジネスコードの変更が不要**
- モックコードとビジネスコードは完全に分離され、本番ビルドに含まれない
- 同じ handler を開発とテストの両方で使い回せる
- REST と GraphQL に対応
- DevTools の Network パネルにリクエストが表示され、デバッグしやすい
:::

## 従来のアプローチの問題点

従来のモック手法は**モックロジックがビジネスコードに侵入している**：

```txt
┏━━━━━━ 手法1：サーバー側での条件分岐 ━━━━━━━━━━━━━━━━━━━━━━━━┓
┃                                                      ┃
┃  app.get("/api/users", (req, res) => {               ┃
┃    if (process.env.MOCK) return res.json(mockData);  ┃  ← モックがビジネスコードに混在
┃    return res.json(await db.query(...));             ┃    消し忘れると本番に影響
┃  });                                                 ┃
┃                                                      ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

┏━━━━━━ 手法2：フロントエンド側での条件分岐 ━━━━━━━━━━━━━━━━━━━━┓
┃                                                       ┃
┃  const users = isDev                                  ┃
┃    ? mockUsers                                        ┃  ← コンポーネント内にモック判定が散在
┃    : await fetch("/api/users");                       ┃    リクエストごとに修正が必要
┃                                                       ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
```

MSW は**ネットワーク層**でインターセプトするため、ビジネスコードは `fetch("/api/users")` と書くだけで、モックされていることを意識しない：

- **開発時**：バックエンド API が未完成でもフロントエンド開発を進められる。リリース前にモック起動コードを削除するだけ
- **テスト時**：`fetch` や axios のモック、偽データの注入が不要。テストの挙動が本番環境と**完全に一致**する

```txt
              fetch("/api/users")
  ┌─────────────────────┐    ┌─────────────────┐    ┌─────────────────┐
  │                     │    │                 │    │                 │
  │  フロントアプリ/テスト │    │  MSW インターセプト層│─✕─▶│    サーバー       │
  │                     │    │                 │    │                 │
  │  fetch() を普通に発行  │◀── │  Handler にマッチ  │    │  到達しない       │
  │  モック判定は不要      │──▶ │  モックレスポンスを返却│    │                 │
  │  コード変更なし        │    │                 │    │                 │
  │                     │    │  ブラウザ:        │    │                 │
  │                     │    │ Service Worker  │    │                 │
  │                     │    │  テスト:          │    │                 │
  │                     │    │   Node.js       │    │                 │
  │                     │    │   インターセプター  │    │                 │
  └─────────────────────┘    └─────────────────┘    └─────────────────┘
```

## インストール

```zsh
% npm install msw --save-dev
% npx msw init ./public --save    # ブラウザ側のみ必要（下記を参照）
```

::: tip `msw init` で何が生成されるのか？
`public/` 配下に `mockServiceWorker.js` が生成される。これは MSW の Service Worker インターセプトスクリプトであり、**編集する必要はない**。自分が書く mock handler とは無関係である
:::

## 基本的な使い方

3ステップで導入：(1) Handler を定義 → (2) インターセプトインスタンスを作成 → (3) アプリに接続。詳細は[基本的な使い方](./usage)を参照
