# API 通信方式

## 数据请求方式

| 维度       | REST                                 | [GraphQL](/programming/web-backend/graphql/) | [tRPC](/programming/web-backend/trpc/) |
| ---------- | ------------------------------------ | -------------------------------------------- | -------------------------------------- |
| 接口定义   | URL + HTTP 方法                      | Schema + 单一端点                            | TypeScript 类型推导                    |
| 数据获取   | 服务端决定返回结构                   | 客户端精确指定所需字段                       | 客户端直接调用服务端函数               |
| 类型安全   | 需要额外工具（OpenAPI）              | Schema 自带类型（可 codegen）                | **零配置端到端类型安全**               |
| 多语言支持 | 任意语言                             | 任意语言                                     | 仅 TypeScript                          |
| 缓存       | HTTP 原生缓存                        | 需额外方案                                   | 依赖 TanStack Query                    |
| 适用场景   | 公开 API、资源结构固定、微服务间通信 | 多资源灵活组合、移动端减少请求               | TypeScript 全栈、前后端同一团队        |

---

## 实时通信选型

| 维度       | HTTP 轮询                  | [SSE](/programming/web-backend/sse/) | [WebSocket](/programming/web-backend/websocket/) |
| ---------- | -------------------------- | ------------------------------------ | ------------------------------------------------ |
| 通信方向   | 客户端主动请求             | 服务端 → 客户端（单向）              | 全双工（双向）                                   |
| 协议       | http:// / https://         | http:// / https://                   | ws:// / wss://（需 101 升级）                    |
| 数据格式   | 任意                       | 仅文本                               | 文本 + 二进制                                    |
| 连接方式   | 短连接（每次请求新建）     | 长连接（不关闭的 HTTP 响应）         | 长连接（协议升级后保持）                         |
| 头部开销   | 每次带完整 HTTP 头         | HTTP 头（仅首次连接）                | 握手后帧头仅 2~14 字节                           |
| 自动重连   | 不适用                     | 浏览器原生支持                       | 需要手动实现                                     |
| 断点恢复   | 不适用                     | 原生支持（Last-Event-ID）            | 需要自己设计                                     |
| 跨域       | CORS                       | CORS                                 | **无同源限制**（需服务端验证 Origin）            |
| 代理兼容性 | 好                         | 好                                   | 部分代理需额外配置                               |
| 实现复杂度 | 低                         | 低                                   | 中等                                             |
| 适用场景   | 兼容性要求高、实时性要求低 | 通知、日志流、AI 流式输出            | 聊天、游戏、协同编辑                             |

::: warning 怎么选

- 大多数"实时"场景其实只需要 **SSE** — 通知推送、实时数据面板、AI 流式输出都是服务端单向推送，客户端发数据用普通 HTTP 请求即可
- 只有**双方需要随时互发消息**的场景（聊天室、在线游戏、协同编辑）才真正需要 **WebSocket**
- **HTTP 轮询**仅在基础设施不支持长连接或实时性要求极低时使用
:::
