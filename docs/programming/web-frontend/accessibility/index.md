# Accessibility

Web 无障碍 — 让所有用户（包括残障用户）都能正常使用 Web 内容

::: warning 特点

- **法律义务**: 日本 JIS X 8341-3 为基准，EU 的 EAA（2025 年施行）推动义务化，日本 2024 年 4 月施行的改正障害者差别解消法要求民间企业提供合理配慮
- **WCAG 准则**: W3C 制定的国际标准 WCAG 2.2 是事实标准
- **4 原则（POUR）**: Perceivable（可感知）・Operable（可操作）・Understandable（可理解）・Robust（健壮性）
- **开发落地**: 语义化 HTML、WAI-ARIA、键盘操作、色彩对比度是主要实现要点
  :::

```
WCAG 达标等级

  Level A    ── 最低限（不满足则大量用户无法使用）
  Level AA   ── 标准目标（日本企业/公共站点的实质基准）
  Level AAA  ── 最高（通常不要求全页面达标）
```

::: tip 日本实务要点
**JIS X 8341-3:2016** 对应 WCAG 2.0 Level AA。多数日本企业以 WCAG 2.1 AA 为目标。2024 年 4 月施行的改正障害者差别解消法，将合理配慮的提供从"努力义务"升级为"法定义务"
:::

## 基础概念

| 概念         | 说明                                                                            |
| ------------ | ------------------------------------------------------------------------------- |
| 语义化 HTML  | 正确使用有意义的 HTML 元素，让辅助技术能理解页面结构（[详见](./semantic-html)） |
| WAI-ARIA     | 补充 HTML 无法传达的信息的属性体系（[详见](./aria)）                            |
| 键盘操作     | 不用鼠标也能访问所有功能（[详见](./keyboard)）                                  |
| 表单         | 将标签、错误、验证信息正确传达给辅助技术（[详见](./form)）                      |
| 测试         | 自动测试 + 手动测试 + 屏幕阅读器，三层保障质量（[详见](./testing)）             |
| 颜色与对比度 | 文本与背景的对比度 4.5:1 以上（AA 基准）                                        |
| 响应式       | 200% 缩放时内容不缺失，reflow 正常工作                                          |
| 替代文本     | 为图片设置合适的 `alt` 属性，装饰性图片用 `alt=""` 跳过                         |

## 基本使用

无障碍对应的基本是**正确书写语义化 HTML**。很多问题仅靠语义化 HTML（[详见](./semantic-html)）就能解决

不够的部分用 WAI-ARIA（[详见](./aria)）补充，再处理键盘操作（[详见](./keyboard)）和表单（[详见](./form)）的对应，最后通过测试（[详见](./testing)）确认质量

### Checklist

```
✅ 所有图片都有合适的 alt
✅ 标题层级正确（h1 → h2 → h3，不跳级）
✅ 表单输入都有关联的 <label>
✅ Tab 键能访问所有可交互元素
✅ 焦点指示器可见
✅ 没有仅靠颜色传达信息
✅ 文本对比度 4.5:1 以上
✅ 页面设置了 lang 属性
✅ 错误信息与输入框有程序关联
```
